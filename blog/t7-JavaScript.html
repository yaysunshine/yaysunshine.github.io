What's The Difference?

JavaScript vs. Ruby

23 January 2015

A whole lot of things! JavaScript and Ruby are two totally different languages. Ok well except for the fact that they are both object oriented languages. And for the fact that I have loved learning both languages.

Really though, the syntax is totally different. I feel like JavaScript is even further from English than Ruby. Just check this out.

WTH his the semicolon at the end of everything? At least Ruby ends things with "end". Oh well. I mean I really am not so frustrated. JavaScript does amazing things.

So I'm going to talk about terms in each language.

Why? Because as with any new language, you have to familiarize yourself with the terms, the vocabulary.

How do you learn vocabulary? Ok well think. You know that dairy product that goes on pizza? Bet you thought of "cheese" right? See! So you saw a definition and thought of a word. Now we want to see what that word with that definition is in Spanish: "queso". Now you know "queso" in Spanish is "cheese" in English.

Basically you learned a word in a new language by comparing it to a word with the same meaning in the language you do know. Awesome.

Let's do that with "console.log();" in JavaScript and "puts", "print", and "p" in Ruby.

Puts, print, and p in Ruby all display your code in like a user-friendly way as opposed to a coder way. Puts, print, and p in Ruby are different from each other in simple ways, but I'm not going to get into it here. Focus.

JavaScript's console.log(); does the same cool display thing.

Why does this matter? I'm learning a language!!! That's how I operate. If you don't do what I did with cheese, let me know how you learn a new language.

Nice and short blog. Yay.
