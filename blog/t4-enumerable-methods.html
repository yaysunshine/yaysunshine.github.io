Dot Map

.map

11 Dec 2014

Enumerables. Say it again. (Or don't, but I had to say the word out loud a few times to understand how to pronounce it).

E - nu - mer - a - bles are Ruby methods that enumerate, or iterate. It - er - ate. (I'm doing this [bad] phonetic style for myself mostly, but yay if you like it, too). Iteration is simple. Ready -- it means GO OVER. Iterating through data structures is going over each element in the data structure.

If you knew that, PROPS. That had me confused for a while. Now to the topic: .map. This is an enumerable method. Enumerable methods iterate.

Since it's an enumerable method, we know one thing about this .map method: it will go over each element in the data structure.

This method basically: 1.) creates a new array (so, it's a non-destructive method, cool); 2.) makes the same operation happen to each object (because, remember, it's iterating). It also uses a block for an array, which is just the stuff within curly braces {}. We'll see what .map does to an array.

Say we have array named favorite_numbers:

favorite_numbers = [10, 100, 5, 4]

Say we want to make each element ten times greater than the current value. Because what's coding if not a bunch of *awesome* word problems? So we could use .map to do this. Here's how:

favorite_numbers.map { |elements| elements * 10 }

So, favorite_numbers is our array. We are calling .map on the array. The word inside the pipes || is a placeholder that represents the elements within the array. Since it's just a placeholder, you could put any word, like "x" or "number" or "coffee" or "sleep". To the right of the pipes, we are saying that we want each element within the array to be ten times greater than its current value...aka, multiply each by 10.

You got this. So, this is our new array, right?

=> [100, 1000, 50, 40]

Yes!! Same elements. Just modified. And in a new array (because .map is non-destructive).

But really, why even use map?

Because it makes life easy! We can do one operation to each and every element in the data structure. Instead of having to change elements within a data structure individually, now we can say: heyyyy man just multiply (or divide or add) alllll of these. Please. And thank you.
