Classes and IDs

Sorry not everything can have a witty title

30 Nov 2014

HTML is a tedious language. Beautiful, but tedious. CSS, its BFF, helps decrease the repetition with classes and IDs.

The biggest difference with classes and IDs is that a class attribute can be assigned to multiple elements within an HTML document whereas an ID is assigned to a single element. For instance, if you, the magical web developer, want to make multiple pictures all the same size, you could assign a class to the divs that the images are sitting in in the HTML. You would then go to the CSS and edit the class with width, height, position, display, etc. This is better than going into each individual div and assigning them the same values. It cuts the repetition.

What if you want one picture a different size? You assign that div an ID. This ID will override the class. As the term suggests, it gives the div a singular identification that no other div can have. The edition of the div in CSS will only apply to the one element that received the ID.
