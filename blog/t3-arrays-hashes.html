Array We Go

Arrays and Hashes

6 Dec 2014

Ruby is my first coding language, my baby steps, my rock from which to build. The only problem I have always had (up until this week) with Ruby, is its arrays. Like what. What does that even mean. Do I display an array to go play in Bombay? Am I "array"nging something? WhAt Do I dO.o?

So beyond all that confusion, arrays are literally lists. They are just lists. Of things. That's it. The catch: the first item on the list is the "zeroth" item. Not the first, the zeroth. For instance, let's name an array "week_days":

week_days = []

Note the syntax: array, then equal sign, then brackets. Right now this array is empty. Let's fill it with...oh how about week days!

week_days = ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday"]

Again, syntax note: the items within an array are surrounded by quotes and separated by commas. Right now, Monday is in the zeroth slot, Tuesday in the first, Wednesday in the second, and so on. If we want Friday, we put:

week_days[4] = "Friday"

Arrays are time savers. If we wanted to make the weeks days all lowercase, we would simply code for that on the array week_days instead of changing each day individually. It is also possible to add, or concatenate, two arrays:

week_days = ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday"] week_ends = ["Saturday", "Sunday"] week_days + week_ends = third_array

Want to guess what the third_array looks like?

third_array = ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday"]

AWESOME! Now, hashes. Hashes, like arrays, store objects, but they need a KEY and a VALUE. Because of their different needs, hashes have a different syntax from arrays:

hash = { key => value }

Why even bother? Well if you need to keep track of what Spongebob and Patrick's last names, you need to assign the key (Spongebob / Patrick) to the value (last name / last name). Let's call this hash awesome_show:

awesome_show = { "Spongebob" => "SquarePants" "Patrick" => "Star" }

Done! Thanks for reading.
