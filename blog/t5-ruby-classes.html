Class-y Lady

3 January 2015

HAPPY NEW YEAR

Howdy. Let's talk about classes shall we.

A class is a thing. It's a thing that groups a bunch of methods (you know those other things that get def-ined).

Let's think of a class like a human. Our awesome human here has bones, muscles, legs, hands, fingers, arms, eyes, ears, mouth, noes...you know like human things. These "human things" are the equivalent of methods. And "human things" are alllll a part of the human. The awesome human is the class.

Ok I hope you get that abstraction. Tweet me if not. (I don't bite, and I follow back!) So now that we have some abstract visual down, let's look at some real syntactiful code.

You know a class is a class when it has the word class in front of it followed by a capital letter:

class AwesomeHuman

See. You got it. Now we want to give our human class some "human things" (methods, remember?). Like a name! We do this by defining a method within the class. The first method always has to be initialized:

class AwesomeHuman

def initialize

Your initialize method can take arguments. We're going to pass the argument name. We then make this name an instance variable. We can manipulate an instance variable outside the method in which it is created. This is handy if we want to use the value of name later on in the code. I want to write a code that tells me how many names my AwesomeHuman has:

class AwesomeHuman

def initialize(name)

@name = name

end

def name

@name.count

end

end

Make sure to end your class! Just this code will not print anything to the screen. We need to test it:

test = AwesomeHuman.new(["Catie", "Kennedy"])

p test.name

I am using sublime and have saved this file as: AwesomeHuman.rb

So now if we run this code in terminal (if you're on a mac, or powershell if you're on a pc):

ruby AwesomeHuman.rb

will print out

2

A quick recap. In our test, we passed the class an array with 2 words, so in that test our AwesomeHuman has 2 names. Let's add another test in our sublime AwesomeHuman.rb file:

testtwo = AwesomeHuman.new(["c", "h", "o", "c", "o", "l", "a", "t", "e"])

p testtwo.name

If we run this code in terminal:

ruby AwesomeHuman.rb

will print out:

2

9

The 2 is from test and the 9 is from testtwo. Note how we p-rinted out the numbers by calling our name method on our test name. Whatever you pass, we have a method within our AwesomeHuman that will return the number of elements (in our case words or letters) in AwesomeHuman's name.

In conclusion, we can make a class have instance variables and instance methods. If a class is the human, instance variables are...like blood. Can be found and used everywhere in the body. And methods are like limbs.

~ Catie Kennedy ~
